COLOR PARSLEY REGEX 0.1.2



GUIDE:

  • GROUP 1: by itself takes 0-255 as INT and must be terminated with a comma or space. By itself generates greyscale by applying the value to all three r g b and alpha at 1. Leading rgb(a) and parenthesis are optional.
  • GROUPS 1 thru 4: G1-3 take 0-255 as INT, must be separated with a comma or space, G4 takes an INT percentage if terminated with % or a float 0 to 1. Leading rgb(a) and parenthesis are optional.
  • GROUPS 5 thru 8: takes three or four digit HEX string. Hash is optional. #afc or 1234. May have an optional space at the end only.
  • GROUP 9: By itself takes a two digit HEX string. Hash is optional. #ab By itself generates greyscale by applying the value to all three r g b and alpha at 1. I.e. #ab becomes ababab
  • GROUPS 9 thru 12: takes six or eight digit HEX string. Hash is optional. #abcdef or 12345678.
  • GROUPS 13 thru 17: "catch all" for various spaces. G13 takes a 3 to 5 character text string identifyer such as lch. The first character can NOT be a-zA-Z but not r or R. Parenthesis are optional. Values can be anything from INT to percentage to float, there is minimal verification. Each value must be separated with a comma or space, except the last (alpha) which can use comma, slash, or a space.


REGEX STRING 0.1.2

Here is the regex string for parsing various types of color strings with validation for "rgb()" and hex strings. Minimal checking for hsl/xyz/lchuv etc. type strings. For clarity it is broken into three pieces per the above flowchart. 


// First section, for rgb() as 
/(?:^(?:rgba?|)\(?\s?(?:(25[0-5]|2[0-4]\d|(?:100|\d{0,2}\.?\d{1,22})%?)(?:, ?| ))(?:(25[0-5]|2[0-4]\d|1?\d{1,2})(?:, ?| )(?:(25[0-5]|2[0-4]\d|1?\d{1,2}))(?:(?:,| \/| )\s?(?:(100%|\d?\d%|[01]?\.\d{1,22}|[01])))?(?: \)| |))?(?!\S)|

// Second section, for hex strings, either:  abc or aabbcc  (or defa or ddeeffaa)
^(?:#|0x|)(?:(?:([\da-f])([\da-f])([\da-f])([\da-f])?)(?!\S)|(?:([\da-f]{2})(?:([\da-f]{2})([\da-f]{2})([\da-f]{2})?)?))(?!\S)|

// Third section, for color(sRGB 1 1 1 / 1)

// Fourth section, for hsl, hwb, lch, and anything as in stuvw(1,1,1,1)
^(?:([^r\d\W][a-z]{2,5})\(?\s?((?:\d{0,3}\.|)\d{1,22}%?)(?:, ?| )((?:\d{0,3}\.|)\d{1,22}%?)(?:, ?| )((?:\d{0,3}\.|)\d{1,22}%?)(?:(?:,| ?\/) ?((?:\d{0,3}\.|)\d+%?))?)(?: \)| |))$/i

<alphavalue>
If the value is internally represented as an integer between 0 and 255 inclusive (i.e. 8-bit unsigned integer), follow these steps:

- Let alpha be the given integer.
- If there exists an integer between 0 and 100 inclusive that, when multiplied with 2.55 and rounded to the closest integer (rounding up if two values are equally close), equals alpha, let rounded be that integer divided by 100.
- Otherwise, let rounded be alpha divided by 0.255 and rounded to the closest integer (rounding up if two values are equally close), divided by 1000.
- Return the result of serializing rounded as a <number>.
- Otherwise, return the result of serializing the given value as a <number>.


14.2 Simple alpha compositing

Graphics elements are blended into the elements already rendered on the canvas using simple alpha compositing, in which the resulting color and opacity at any given pixel on the canvas is the result of the following formulas:

Er, Eg, Eb    - Element color value
Ea            - Element alpha value
Cr, Cg, Cb    - Canvas color value (before blending)
Ca            - Canvas alpha value (before blending)
Cr', Cg', Cb' - Canvas color value (after blending)
Ca'           - Canvas alpha value (after blending)
Ca' = 1 - (1 - Ea) * (1 - Ca)

Premultiplied colors
Cr' = (1 - Ea) * Cr + Er
Cg' = (1 - Ea) * Cg + Eg 
Cb' = (1 - Ea) * Cb + Eb


Straight not Premultiplied
Cr' = (1 - Ea) * Cr*Ca + Er*Ea 
Cg' = (1 - Ea) * Cg*Ca + Eg*Ea 
Cb' = (1 - Ea) * Cb*Ca + Eb*Ea
